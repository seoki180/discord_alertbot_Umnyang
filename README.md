# 윰냥이 공지 읽음 알림봇

Discord 서버의 공지 메시지를 등록하면, 24시간 뒤에 ✅ 이모지를 누르지 않은 멤버들에게 DM으로 알림을 보내는 봇입니다.

---

## 주요 기능

- 슬래시 커맨드 `/공지등록`으로 공지 메시지를 읽음 추적 대상으로 등록
- ✅ 이모지 반응을 읽음 처리로 인식 (추가/취소 모두 실시간 반영)
- 등록 후 24시간 뒤, 미읽음 멤버에게 DM 발송
- 관리자에게 미읽음 현황(목록 포함) DM 발송
- 봇 재시작 시 타이머 자동 복원
- 특정 역할을 가진 멤버만 대상으로 필터링 가능

---

## 설치 및 실행

### 1. 의존성 설치

```bash
npm install
```

### 2. 환경변수 설정

프로젝트 루트에 `.env` 파일을 생성하고 아래 값을 입력합니다.

```env
DISCORD_TOKEN=봇_토큰
NOTICE_CHANNEL_ID=공지_채널_ID
ROLE_10TH=대상_역할_ID
ADMIN_ID=관리자_유저_ID
```

| 변수명 | 설명 |
|---|---|
| `DISCORD_TOKEN` | Discord 개발자 포털에서 발급한 봇 토큰 |
| `NOTICE_CHANNEL_ID` | 공지가 올라오는 채널 ID |
| `ROLE_10TH` | 읽음 확인 대상이 되는 역할 ID (이 역할을 가진 멤버만 추적) |
| `ADMIN_ID` | 미읽음 현황 DM을 받을 관리자 유저 ID |

### 3. 봇 실행

```bash
node bot.js
```

---

## Discord 개발자 포털 설정

봇이 정상 동작하려면 다음 권한(Intents)이 활성화되어 있어야 합니다.

- **Server Members Intent** — 멤버 목록 조회에 필요
- **Message Content Intent** — 메시지 내용 읽기에 필요

> Discord 개발자 포털 → 봇 설정 → Privileged Gateway Intents에서 활성화

## 사용법

### `/공지등록` 커맨드

과거에 올라온 공지 메시지를 읽음 추적 대상으로 등록합니다.

```
/공지등록 메시지id: <메시지_ID>
```

**메시지 ID 복사 방법:**
1. Discord에서 개발자 모드 활성화 (설정 → 고급 → 개발자 모드)
2. 공지 메시지에 우클릭 → **ID 복사**

**실행 결과:**
- 이미 ✅를 누른 인원 수 표시
- 등록 시점으로부터 24시간 뒤 자동 미읽음 체크 예약

> 이 커맨드는 `ROLE_10TH` 역할을 가진 멤버만 사용 가능합니다.

---

### 읽음 처리 (✅ 이모지)

공지 메시지에 ✅ 이모지를 누르면 읽음 처리됩니다.
이모지를 제거하면 읽음 취소로 처리됩니다.

---

### 24시간 후 자동 알림

공지 등록 후 24시간이 지나면:

1. 공지 채널 멤버 중 대상 역할을 가진 멤버 전체를 조회
2. ✅ 를 누르지 않은 멤버에게 DM 발송
3. 관리자에게 미읽음 멤버 목록 DM 발송

---

## 데이터 저장

공지 추적 정보는 프로젝트 루트의 `data.json` 파일에 저장됩니다.
봇이 재시작되어도 등록된 공지와 남은 타이머가 자동으로 복원됩니다.

```json
{
  "메시지ID": {
    "guildId": "서버 ID",
    "channelId": "채널 ID",
    "content": "공지 내용 앞 100자",
    "postedAt": 1234567890000,
    "notifiedAt": null,
    "reactedUsers": ["유저ID1", "유저ID2"]
  }
}
```
